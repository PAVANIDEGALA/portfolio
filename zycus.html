CONTEXT

The user has a legal document in hand. He needs to know what are the aspects that should be concerned about before signing it. In an offline world he shall take it to a legal consultant. This product help him compare the major aspects against a standard template and evaluate the same.


PROBLEM STATEMENT

Create an interface for the user to compare 2 documents.
Document 01: User uploaded legal document.
Document 02: System Template
The system should be able to compare both the documents and evaluate the risks/issues of the uploaded document.


PROBLEM ANALYSIS

PROBLEM : When the user has a legal document , he should be concerned about the risks/issues in the document and before signing it,he should be aware of all the consequences.

IMPACT : Sometimes, he may miss important details which may cause a major problem in the future. The product we create should be able to solve this problem for the user. Example : If an agreement says that you should not work for any other company while working with them and you haven’t read that and applied to several internships too. You have to facethe consequences later. Here, the actual problem is that the user should be concerned about particuar aspects before signing it, which means he should know what are the risks and the issues of the contract. There is a possibility of missing out important things if he checks the agreement by himself.
